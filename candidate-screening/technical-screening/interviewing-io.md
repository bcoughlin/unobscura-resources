# Interviewing.io

**Category:** Technical Screening  
**Format:** Anonymous voice-only technical interviews  
**Assessment:** ✅ More Neurodivergent-Friendly

---

## What They Do

Interviewing.io conducts anonymous technical interviews where candidates are identified only by voice (no video). The platform attempts to reduce bias by removing visual cues.

---

## Why This Is Better

**Anonymity Reduces Some Bias:**
- No visual bias based on appearance
- No facial expression analysis
- Voice-only can reduce sensory overwhelm for some
- Removes some social performance pressure

**Human Interviewers:**
- Real people, not AI
- Can adapt to different communication styles
- May be more flexible than automated systems

**Philosophy:**
- Founded by someone who experienced interview bias
- Actively trying to make technical hiring more fair
- Research-oriented approach

---

## Still Challenging

**Live, Synchronous Format:**
- Real-time interview (not asynchronous)
- Still creates time pressure
- Processing differences not fully accommodated

**"Think Aloud" Expected:**
- Need to verbalize your thinking while coding
- Unnatural for many neurodivergent developers
- Bottom-up thinkers need more time to explain

**Timed Assessments:**
- Usually 45-60 minute sessions
- Speed still matters

---

## What to Expect

**Format:**
- Voice-only call (no video)
- Coding problem(s) in shared editor
- 45-60 minutes typically
- Interviewer asks questions and evaluates

**Evaluation:**
- Technical correctness
- Problem-solving approach
- Communication (verbal explanations)
- Code quality

---

## Accommodation Strategies

**Before the Interview:**
- Ask if asynchronous option exists
- Request written explanation option
- Inquire about extended time
- Mention you process information differently if comfortable

**During the Interview:**
- "I need a moment to think" is okay
- You can summarize approach before or after coding
- Ask clarifying questions
- Comment your code thoroughly as substitute for verbal

**If Stuck:**
- Explain what you'd research
- Demonstrate debugging approach
- Show problem-solving process even if incomplete

---

## Alternative Approaches to Propose

"I work better asynchronously - could I complete a take-home challenge?"

"Could I code privately and then walk through my solution?"

"Would you consider reviewing my GitHub portfolio instead?"

---

## Bottom Line

**Best option in technical screening category**, but still has the fundamental issues of:
- Live performance
- Verbal communication requirements  
- Time pressure

If you must do a technical screening platform, this is the most accommodating of the major options.
